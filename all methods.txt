• Transform conceptual model along with
reconciled data into atomic elements
• Output is Logical Schema

Modeling Data in Business
Model is simplified version of a concept, phenomenon, relationship or
an aspect of the real world
• E-R Model is mainstream approach for conceptual modeling
• Used as communication tool between DB designer & end user during analysis phase
• Enhanced version is later used for physical design of DB
E-R Model is logical representation of organizational data or business area using entities
(data) and relationships between entities (associations)

EER – Sub Type
Subgrouping of entities into an entity type that is meaningful to the
organization
• Entities should share common attributes or relationships
• Attributes & Relationships must be different/distinct from other subgroups
• Generally applied where most of the attributes are common except only few
• Example: Snake, Sparrow, Pigeon, Cat, Dog
EER – Super Type
Generic entity type that has relationship to one or more sub types
• Holds common or shared attributes to all participating sub types (entity types)
• Relationships of Super Type to other Entity Types belong to all sub types (entity types)
• Example: Animal, Vehicle

Relational Algebra - Union
Two relations R and S defines a relation that contains all the
tuples of R, or S, or both R and S, duplicate tuples being eliminated. R and
S must be union-compatible.
• Concatenate tuples from R & S
• Schema of both relations should match
• Should have same number and name of attributes in Selection or Projection
• This property is called Union-Compatible
Relational Algebra - Union
σ predicate (R) σ predicate (S)U
Π attributes(R) Π attributes(S)U

Third Normal Form (3-NF)
• Relation is in 2-NF
• No non-prime attribute is dependent on any other non-prime attribute (called Transitive
Functional Dependency)
A B
B A
B C
A C
A determines B  B is dependent on A
B doesn’t determines A  A is not dependent on B
B determines C  C is dependent on B
A determines C because A determines B and B determines C
A  C is transitive dependency
